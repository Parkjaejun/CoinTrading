## 🚀 GUI 기반 주요 기능

### 🎯 완전한 시각적 제어
- **실시간 대시보드**: 포지션, 차트, 잔고 한눈에 확인
- **원클릭 제어**: 거래 시작/중지, 포지션 관리 버튼으로 간편 제어
- **실시간 차트**: 100개 데이터 포인트 기반 가격 움직임 시각화
- **다크 테마**: 눈의 피로를 줄이는 전문적인 트레이딩 인터페이스

### 🔄 듀얼 전략 자동매매
- **롱 전략**: EMA 골든크로스 기반 상승장 매매
- **숏 전략**: EMA 데드크로스 기반 하락장 매매
- **실시간/가상 자산**: 손실 시 자동으로 가상자산 전환
- **트레일링 스탑**: 수익 보호를 위한 동적 손절매

## 🖥️ GUI 실행 방법

### 1. 필수 준비사항
```bash
# 1. 라이브러리 설치
pip install PyQt5==5.15.9
pip install pyqtgraph==0.13.3
pip install psutil==5.9.6

# 2. 프로젝트 구조 확인
CoinTrading/
├── gui/
│   ├── __init__.py              # 빈 파일 생성 필요
│   ├── main_window.py           # 메인 GUI
│   ├── widgets.py               # GUI 위젯들
│   └── settings_dialog.py       # 설정 다이얼로그
├── run_gui.py                   # GUI 런처
└── config.py                    # API 설정
```

### 2. GUI 실행
```bash
# 방법 1: 런처 사용 (권장)
python run_gui.py

# 방법 2: 직접 실행
python gui/main_window.py
```

### 3. 초기 설정 확인
실행 시 다음 항목들이 자동으로 확인됩니다:
- ✅ PyQt5, pyqtgraph, psutil 라이브러리
- ✅ config.py의 API 설정
- ✅ OKX API 연결 상태

## 🤖 자동매매 전략 시스템

### 📈 **롱 전략 (알고리즘 1번)**

#### 🎯 전략 개요
- **시간프레임**: 30분봉
- **기본 조건**: 상승장 (150EMA > 200EMA)
- **레버리지**: 10배
- **트레일링 스탑**: 고점 대비 10%

#### 📊 진입/청산 로직
```python
# 진입 조건 (골든크로스)
상승장 확인: 150EMA > 200EMA
진입 신호: 20EMA가 50EMA를 상향 돌파

# 청산 조건 (데드크로스)
청산 신호 1: 20EMA가 100EMA를 하향 돌파
청산 신호 2: 트레일링 스탑 (고점 대비 10% 하락)
```

#### 🛡️ 위험 관리
- **손실 한계**: 고점 대비 20% 하락 시 거래 중단
- **자산 전환**: 실제 자산 → 가상 자산으로 자동 전환
- **재진입**: 가상자산으로 30% 회복 시 실제 거래 재개

### 📉 **숏 전략 (알고리즘 2번)**

#### 🎯 전략 개요
- **시간프레임**: 30분봉
- **기본 조건**: 하락장 (150EMA < 200EMA)
- **레버리지**: 3배
- **트레일링 스탑**: 저점 대비 2%

#### 📊 진입/청산 로직
```python
# 진입 조건 (데드크로스)
하락장 확인: 150EMA < 200EMA
진입 신호: 20EMA가 50EMA를 하향 돌파

# 청산 조건 (골든크로스)
청산 신호 1: 100EMA가 200EMA를 상향 돌파
청산 신호 2: 트레일링 스탑 (저점 대비 2% 상승)
```

#### 🛡️ 위험 관리
- **손실 한계**: 고점 대비 10% 하락 시 거래 중단
- **자산 전환**: 실제 자산 → 가상 자산으로 자동 전환
- **재진입**: 가상자산으로 20% 회복 시 실제 거래 재개

### 🔄 **듀얼 자산 시스템**

#### 💡 작동 원리
```
실제 자산 (A) ←────→ 가상 자산 (B)
     ↓                    ↓
  실제 거래            가상 거래 (검증)
     ↓                    ↓
 손실 한계 도달 ─────→ 가상으로 전환
     ↓                    ↓
 거래 중단            가상에서 회복
     ↓                    ↓
 대기 상태 ←────── 실제로 복귀
```

#### 🎛️ GUI에서의 표시
- **실제 모드**: 녹색 표시, "REAL TRADING" 라벨
- **가상 모드**: 주황색 표시, "VIRTUAL TRADING" 라벨
- **자동 전환**: 손실 한계 도달 시 자동으로 모드 변경
- **복구 조건**: 가상에서 설정 수익률 달성 시 실제 모드 복귀
